# Review

Use this skill when asked to review code, config, or a proposed change.

- Lead with findings ordered by severity.
- Include file and line references when available.
- Focus on bugs, regressions, security risks, missing tests, and operational hazards.
- Keep summaries brief and secondary to findings.
- If no issues are found, say so explicitly and mention residual risk or testing gaps.
